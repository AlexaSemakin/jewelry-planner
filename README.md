# Jewelry Production Planner

Веб-приложение для планирования и оптимизации производства авторских ювелирных
изделий с распределёнными исполнителями и логистикой между городами.

## Стек

- **Backend**: Python 3.11, FastAPI, SQLAlchemy 2, Pydantic v2
- **БД**: PostgreSQL 16
- **Оптимизация**: Google OR-Tools (CP-SAT) — модель RCPSP с ограниченными ресурсами
- **Frontend**: React 18 + Vite + TailwindCSS + Recharts
- **Запуск**: Docker Compose

## Быстрый старт

```bash
docker compose up --build
```

После сборки:

- UI: http://localhost:5173
- API (Swagger): http://localhost:8000/docs
- БД: localhost:5432 (jewelry / jewelry / jewelry)

При первом запуске БД автоматически наполняется тестовыми данными
(города, исполнители, маршруты, несколько демо-заказов).

## Что есть в системе

- **Менеджер**: режим оценки заказа — стоимость, срок, реализуемость.
- **Технолог**: конструктор техпроцесса, справочники исполнителей и маршрутов,
  запуск заказов в работу, отслеживание выполнения.
- **Оптимизация**: подбор исполнителей, маршрутов и расписания с учётом
  ограниченного числа курьеров, рабочих календарей и сроков по всем активным
  заказам одновременно.
- **Объяснимость**: для каждого этапа показываются альтернативы и причина выбора.
- **Поведение при невыполнимости**: явно сообщается, какой минимальный срок возможен.
- **Имитация транспортных интеграций**: модуль `TransportAdapter` с тестовым
  провайдером; реальный коннектор подключается заменой одного класса.
